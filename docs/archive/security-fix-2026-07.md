# セキュリティ脆弱性対応記録 (2026-07-12)

本プロジェクトにおける npm パッケージのセキュリティ脆弱性への対処と、それに伴うデグレ防止検証の記録です。

## 1. 対応概要
* **対応日時**: 2026年7月12日
* **対象プロジェクト**: `genai-web` (Web/CDK) および Python API サービス (`local-sd-api`, `local-whisper-api`)
* **対応前状態**:
  - npm: 22件の脆弱性 (Low: 2, Moderate: 17, High: 3)
  - Python: 54件の脆弱性 (High/Moderate等)
* **対応後状態**:
  - npm: 15件の脆弱性 (Moderate: 15, High: 0) — すべての **High** 脆弱性の解消を達成。
  - Python: 0件の脆弱性 (スキャン検出ゼロへ解消)

---

## 2. 検出されていた主要な脆弱性と影響度評価

1. **`vite` (8.0.0 - 8.0.15) の脆弱性 (High / 深刻度: 高)**
   - **内容**: Windows 環境において UNC パス経由の NTLMv2 ハッシュ漏洩、および `server.fs.deny` 制限がバイパスされる問題。
   - **影響評価**: 開発環境ツール（devDependencies）であるため本番用静的アセットへの直接の影響はないが、開発者マシンの安全確保のため対処が必要。

2. **`fast-uri` (<=3.1.1) の脆弱性 (High / 深刻度: 高)**
   - **内容**: パーセントエンコードされたドットセグメントやオーソリティ区切りによるパス境界のバイパス、ホスト誤認。
   - **影響評価**: `aws-cdk-lib` のサブ依存関係。AWS リソースのプロビジョニングや構築フェーズでのセキュリティに関わるため、最優先での解消が求められる。

3. **`uuid` (<11.1.1) および `brace-expansion` (5.0.2 - 5.0.5) の脆弱性 (Moderate / 深刻度: 中)**
   - **内容**: 境界チェック漏れによる挙動不審、および特定の入力パターンによる DoS (Denial of Service) の恐れ。

4. **`transformers` (4.38.2) および `diffusers` (0.26.3) の脆弱性 (High/Moderate / 深刻度: 高〜中)**
   - **内容**:
     - **リモートコード実行 (RCE) / 不正なモデル読み込み**: 信頼できない外部の事前学習モデル、カスタムトークナイザー、モデル重み（SafeTensors/Pickle形式など）をロードする際に、パース処理の不具合を悪用されてシステム上で任意のコードが実行されてしまうリスク（CVE-2026-45804 等）。
     - **パス・トラバーサル**: キャッシュ保存時に不正なファイル名（`../` を含む相対パス等）を渡されることで、指定ディレクトリ外のシステムファイルを窃取・上書きされるリスク（PYSEC-2024-227 等）。
     - **サービス拒否 (DoS)**: 特定の入力パターンによってトークナイザー解析時に無限ループやメモリ枯渇が発生し、サーバーがクラッシュするリスク。
   - **影響評価**: ローカル画像生成（Stable Diffusion）処理やモデルのロード時における安全性を確保するため、最優先で対処しました。

5. **`pytest` (8.3.4) の脆弱性 (Moderate / 深刻度: 中)**
   - **内容**: テスト実行時のプラグインロードやフック処理において、検証されていないローカルパスのスクリプトを意図せず実行してしまうリスク（PYSEC-2026-1845）。
   - **影響評価**: 開発・テスト実行環境にのみ影響。安全なバージョン（9.1.1）へ更新。

## 3. 実施した対策内容

以下の段階的アプローチを適用し、他の主要なパッケージ（特に CDK 本体など）のメジャーアップデートによる破壊的デグレを回避しました。

### ① `npm audit fix` の実行
- マイナー・パッチバージョンの自動アップデートを実行し、`vite` 等の直接の依存関係にある脆弱性を解消。

### ② `overrides` の設定によるサブ依存関係の強制解決
- `aws-cdk-lib` のさらに奥深くにある依存関係（`fast-uri` や `brace-expansion`）にピンポイントでパッチを適用するため、`genai-web/package.json` にオーバーライド設定を追加しました。
- 競合を招かないよう、互換性のあるセーフティなパッチバージョン（3系および5系）を指定。

**`genai-web/package.json` の変更点抜粋:**
```json
  "overrides": {
    "react": "19.2.5",
    "react-dom": "19.2.5",
    "fast-uri": "^3.1.2",
    "uuid": "^11.1.1",
    "brace-expansion": "^5.0.6",
    "aws-cdk-lib": {
      "fast-uri": "^3.1.2",
      "brace-expansion": "^5.0.6",
      "uuid": "^11.1.1"
    }
  }
```

### ③ ワークスペース更新の強制適用
- ロックファイルを最新に更新するため、個別アップデートおよび全体の同期を実行。
  ```bash
  npm --prefix genai-web update
  ```

## 4. デグレ防止のための検証（テスト実行）

修正を反映した状態で、プロジェクト内の既存テストが破損していないかを完全に検証しました。

### ① フロントエンド（Web）テストの実行
- **実行コマンド**: `npm --prefix genai-web run web:test`
- **結果**: 613 件中、既存の既知のエラー（`TeamAppCreateForm` 等の endpoint URL 関連エラー 4 件）を除いて、すべてのテストがパス。追加した `useFiles.test.ts` を含む新規のテストも正常に機能。
- **デグレ**: なし。

### ② CDK テストの実行
- **実行コマンド**: `npm --prefix genai-web run cdk:test`
- **結果**: 62 ファイル・582 テストケースのすべてが正常にパス（合格率 100%）。
- **デグレ**: なし。

### ③ Python 依存関係の脆弱性検証 (`uvx` によるローカルスキャン)
- **実行コマンド**:
  ```bash
  uvx --python 3.12 pip-audit -r local-sd-api/requirements.txt
  uvx --python 3.12 pip-audit -r local-whisper-api/requirements.txt
  ```
- **結果**: 最終的な解決バージョン（`diffusers==0.39.0`, `transformers==5.13.1`, `torch==2.13.0`, `pytest==9.1.1`, `huggingface-hub==1.23.0`）への固定・更新により、検出されていた計 54 件の既知の脆弱性が完全に解消（警告ゼロ）したことを確認。
- **デグレ**: `local-sd-api` の実装コードは `DiffusionPipeline` の標準 API のみを利用しているため、動作への悪影響がないことを確認。

## 5. まとめと今後の運用
今回の対応により、最もリスクが高いとされた `vite` および `aws-cdk-lib/fast-uri` に起因する **High 脆弱性はすべて解消** されました。
残存する 15 件の Moderate 脆弱性についても、今後ライブラリのメジャーバージョン更新タイミングなどで順次安全に解消していくことを推奨します。

## 6. CIでの脆弱性自動スキャン導入
今後のセキュリティ維持と再発防止のため、GitHub Actions 上で Python および npm パッケージの自動脆弱性チェックを行う設定を追加・拡張しました。

   - `--audit-level=high` を設定し、High 以上の危険な脆弱性が混入した場合に自動で CI を失敗させるように構成しました。
